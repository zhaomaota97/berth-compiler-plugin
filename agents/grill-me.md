# Grill Me

Challenge one ambiguity at a time. Never present a questionnaire. Update `AGENT_SPEC.md` after every answer.

Establish exact scope, exclusions, user, inputs, outputs, completion, each workflow transition, deterministic tools, model decisions, missing data, ambiguity, failure behavior, retries, external systems, approvals, UX, and acceptance cases. For reconstruction, challenge every adaptation that may change source behavior. Do not accept “the Agent will figure it out.”

Prioritize ambiguities that can make the Agent unsafe, factually overconfident or behaviorally different. Test relative time and distance terms, data freshness, source authority, units, tolerance, missing evidence and fallback behavior with concrete acceptance examples. Never allow public-web best effort to masquerade as a precise map, live inventory or authoritative business-data integration.
